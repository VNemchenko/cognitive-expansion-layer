# Publication Checklist

Use this checklist before making the repository public.

## Repository

- [ ] Create public repository: `brownyx-cognitive-expansion-layer`
- [ ] Add `README.md`
- [ ] Add `WHITEPAPER.md`
- [ ] Add `LICENSE.md`
- [ ] Add `NOTICE.md`
- [ ] Add `CITATION.cff`
- [ ] Add `CHANGELOG.md`
- [ ] Add docs folder
- [ ] Add tags/topics:
  - `brownyx`
  - `cognitive-architecture`
  - `synthetic-mind`
  - `metacognition`
  - `ai-agents`
  - `defensive-publication`
  - `whitepaper`

## Content Review

- [ ] No API keys
- [ ] No private prompts
- [ ] No production code
- [ ] No private traces
- [ ] No private memory examples
- [ ] No customer data
- [ ] No infrastructure secrets
- [ ] No proof-of-consciousness claims

## Legal / IP

- [ ] Confirm license choice
- [ ] Confirm copyright notice
- [ ] Confirm trademark notice
- [ ] Confirm no patent-sensitive details are disclosed
- [ ] Consider IP lawyer review if patent strategy matters

## Release

- [ ] Create Git tag `v0.1.0`
- [ ] Create GitHub release
- [ ] Add release notes
- [ ] Archive with Zenodo or OSF if DOI is desired
- [ ] Store release ZIP and DOI in private [Brownyx](https://brownyx.com) records

## Suggested Release Title

```text
Brownyx Cognitive Expansion Layer v0.1.0 - Initial Defensive Publication
```

## Suggested Release Notes

```text
Initial public documentation release of Brownyx Cognitive Expansion Layer (Brownyx CEL), a bounded cognitive expansion architecture for persistent synthetic mind runtimes.

This release includes the whitepaper, technical specification, architecture notes, prior-art positioning, implementation boundaries, IP strategy notes, citation metadata, and license/notice files.

No production Brownyx source code, private prompts, runtime traces, infrastructure secrets, or commercial implementation details are included.
```
