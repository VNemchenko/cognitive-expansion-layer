# License Options

This repository includes a default `LICENSE.md` using CC BY 4.0 for documentation.

The maintainer may choose a different license strategy before publication.

## Option A: CC BY 4.0

Best if you want the work to be citeable, shareable, and reusable with attribution.

Pros:

- easy to cite;
- compatible with academic/public documentation;
- allows broad distribution;
- requires attribution.

Cons:

- allows commercial reuse of documentation;
- does not prevent independent implementation of the concept.

## Option B: CC BY-NC 4.0

Best if you want to allow non-commercial reuse with attribution.

Pros:

- more restrictive than CC BY;
- discourages commercial republishing of documentation.

Cons:

- not open content in the strongest sense;
- “non-commercial” can be ambiguous;
- still does not prevent independent implementation of the concept.

## Option C: All Rights Reserved

Best if you want public timestamping but minimal reuse rights.

Pros:

- strongest control over documentation text;
- clear reservation of rights.

Cons:

- less friendly for citation/reuse;
- may reduce public spread;
- still does not protect the abstract idea.

## Recommendation

For a defensive publication, use CC BY 4.0 for documentation and keep production code, prompts, scoring, and implementation details private.
